HOW TO UPDATE A GITHUB FORKED REPOSITORY

My question was: how can I update a forked repo in order to view all the latest commits? I found the answer on StackOverflow. I quote it as a note to self. In your local clone of your forked repository, you can add the original GitHub repository as a “remote”. (“Remotes” are like nicknames for the URLs of repositories - origin is one, for example.) Then you can fetch all the branches from that upstream repository, and rebase your work to continue working on the upstream version.

You can also appoint an individual ‘cutter’ to take this role outside of the meeting, but only after brainstorming.
